# Plan: Multistep

## Overview

Upgrade the existing Shared State UI prototype into a **generic AI-driven multi-step workflow engine**.
The system no longer relies on predefined task types and instead supports **fully generic tasks** generated by Gemini. Each interaction is modeled as a **dynamic step** within a workflow, where the model generates the next step based on the previous user input and accumulated state.

The system maintains **complete step history**, allowing backward navigation, session persistence, and state-aware reasoning. Steps can contain **multiple UI inputs** selected from a predefined set of accessible components.

The architecture emphasizes:

* Generic AI task orchestration
* Dynamic multi-step workflows
* Persistent shared state
* Accessible UI rendering
* Structured model outputs
* Step history replay
* Workflow estimation
* Logging and debugging

The final system behaves as an **AI Agent UI**, enabling users to complete complex workflows through incremental interaction.
